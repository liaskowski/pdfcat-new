# Архитектура проекта

## Обзор
Проект состоит из двух основных частей: клиентского приложения на PyQt6 и серверной части на FastAPI.

## Структура и ответственность модулей

### Клиент (Client)
- **`client/main.py`**: Точка входа в клиентское приложение. Инициализация QApplication и запуск главного окна.
- **`client/main_window.py`**: Главное окно (Оркестратор). Отвечает за координацию между различными компонентами интерфейса (поиск, сетка файлов, панель предпросмотра).
- **`client/ui/`**: Набор визуальных компонентов:
    - `file_grid.py`: Отображение списка файлов в виде сетки.
    - `search_bar.py`: Поисковая строка с фильтрами.
    - `preview_panel.py`: Боковая панель с детальной информацией.
- **`client/api_manager.py`**: Слой для взаимодействия с REST API сервера.
- **`client/workers.py`**: Многопоточность (QThread/QRunnable) для фоновых задач.

### Сервер (Server)
- **`main.py` (корень)**: Текущая точка входа FastAPI. Содержит логику API, модели БД и обработку файлов (в будущем планируется перенос в `server/main.py`).
- **`database.py`**: Логика подключения к базе данных SQLite/PostgreSQL и управление сессиями.

## Потоки данных
1. **Действие пользователя**: Пользователь взаимодействует с UI (например, нажимает "Загрузить").
2. **Вызов API**: UI вызывает соответствующий метод в `api_manager.py`.
3. **Обработка на сервере**: FastAPI получает запрос, выполняет бизнес-логику (парсинг PDF, OCR) и сохраняет данные в БД.
4. **Обновление интерфейса**: По завершении запроса клиент получает данные и обновляет UI (через механизмы сигналов PyQt).
