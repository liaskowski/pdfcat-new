# Стандарты кодирования

## Правило 1: Декомпозиция и размер файлов
- Максимальный размер файла не должен превышать **300 строк**.
- Если файл разрастается больше этого лимита, его необходимо разделить на логические модули или вынести часть функционала в `utils/` или `services/`.

## Правило 2: Разделение ответственности (SoC)
- Интерфейс (UI на PyQt) должен быть полностью отделен от бизнес-логики и сетевых запросов.
- Все обращения к API должны быть инкапсулированы в `api_manager.py` или аналогичные сервисные классы.
- Тяжелые операции (обработка файлов, OCR) запрещены в основном GUI-потоке.

## Правило 3: Работа с базой данных
- Все изменения в БД требуют явного вызова `commit()`.
- Использование контекстных менеджеров для сессий БД обязательно для предотвращения утечек соединений.

## Правило 4: Навигация и отображение
- В дереве навигации (если оно используется) отображаются только папки (структура).
- Файлы отображаются исключительно в основной сетке (Grid view) или списке.

## Правило 5: Типизация
- Использование Python Type Hints обязательно для всех новых функций и методов.
- Pydantic модели должны использоваться для валидации всех входящих и исходящих данных API.
